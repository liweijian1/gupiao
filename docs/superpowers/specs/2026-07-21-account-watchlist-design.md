# 邮箱账户与自选股设计

## 目标与边界

为 QuantDesk 增加邮箱加密码注册、登录和每位用户独立的自选股票。未登录用户仍可浏览全部行情、宏观和 AI 面板；仅在尝试加入或移除自选股时要求注册或登录。

本期不包含邮箱验证、找回密码、管理员账号管理、第三方登录或全站访问强制登录。

## 方案

后端使用 SQLite 保存用户、会话和自选股票。密码只使用 bcrypt 哈希（新增 `bcrypt>=4.1,<5` 依赖），浏览器通过同源 `HttpOnly`、`SameSite=Lax` 的会话 Cookie 识别当前用户。会话有效期为 30 天，登录时轮换会话。每次会话查询都从服务端读取当前自选列表，因此同一账号在不同设备登录后可同步自选股票。

SQLite 数据库路径由 `AUTH_DB_PATH` 环境变量指定；未设置时使用后端持久化缓存目录。线上环境必须将该路径置于发布目录之外，避免前端发布切换覆盖用户数据。Cookie 路径由 `AUTH_COOKIE_PATH` 指定，当前部署设为 `/stock-macro/`。当前线上站点是 HTTP，因此 `AUTH_COOKIE_SECURE=false`；启用 HTTPS 后必须设为 `true`。

## 数据模型

`users`

- `id`：整数主键。
- `email`：规范化为小写、唯一。
- `password_hash`：bcrypt 哈希。
- `created_at`：UTC 时间。

`sessions`

- `id`：不可预测的随机令牌；仅存令牌哈希。
- `user_id`：关联用户。
- `expires_at`、`created_at`：UTC 时间。

`watchlist_items`

- `user_id`、`ticker`：联合唯一键。
- `created_at`：UTC 时间，用于按加入顺序展示。

自选记录仅保存股票代码；展示名称、价格和评分继续使用现有股票快照，以避免缓存过期副本。

## API

所有接口位于 `/api/auth` 或 `/api/watchlist`，与既有 AI 和行情接口隔离。

- `POST /api/auth/register`：接收邮箱和密码，创建账户、签发 Cookie 并返回当前用户。
- `POST /api/auth/login`：验证邮箱和密码、轮换会话并返回当前用户。
- `POST /api/auth/logout`：撤销当前会话并清除 Cookie。
- `GET /api/auth/session`：返回当前用户或匿名状态。
- `GET /api/watchlist`：返回当前用户的自选股票代码，未登录返回 401。
- `PUT /api/watchlist/{ticker}`：加入自选，幂等，未登录返回 401。
- `DELETE /api/watchlist/{ticker}`：移除自选，幂等，未登录返回 401。

写操作要求 JSON 请求和同源 `X-Requested-With: QuantDesk` 请求头；后端同时验证 Origin/Referer（存在时）与 `AUTH_ALLOWED_ORIGINS`，降低 Cookie 会话的跨站请求风险。

## 前端体验

顶部命令栏显示匿名状态的“登录”入口，登录后显示当前邮箱与“退出登录”。点击当前股票详情标题旁的星标可加入或移除自选；未登录时该操作打开统一的注册/登录弹窗。

左侧股票发现面板新增“自选”筛选入口，与 A 股、港股、美股范围并列但不改变原有股票搜索和因子筛选状态。自选视图按加入时间展示可用股票；若某个代码暂未出现在当前快照中，保留自选记录并在行情恢复后自动显示。

注册和登录使用同一弹窗，可切换模式。邮箱格式错误、密码不足、邮箱已注册、凭据错误、会话过期和网络失败均显示明确、不可泄露账户存在性的提示。

## 组件与状态边界

- 后端新增独立的认证/自选模块：数据库初始化、密码/会话服务和 FastAPI 路由分离。
- 前端新增 `api/auth.js`、`api/watchlist.js` 与 `useAuth`、`useWatchlist`，负责会话恢复、请求状态和失效处理。
- 账户弹窗只处理表单与认证状态；`App` 持有已登录用户和自选代码集合。
- `EquityDiscoveryPanel` 只接收自选代码、筛选状态与回调，不直接发起认证请求。

## 错误处理与安全

- 邮箱采用 trim 后的小写规范化；密码长度为 10–128 个字符。
- 登录错误统一返回“邮箱或密码不正确”。
- 注册接口对同一邮箱返回“该邮箱已注册”；前端仅在用户主动注册时展示。
- 会话过期或服务端撤销后，前端清空用户和自选状态，并在下一次星标操作时打开登录弹窗。
- 数据库写入使用唯一约束和事务，避免重复加入或并发删除导致错误。
- API 响应不返回密码、哈希或会话令牌。

## 测试与验收

- 后端覆盖注册、邮箱规范化、密码哈希、重复注册、登录失败、会话恢复、退出、跨用户隔离和自选幂等操作。
- 前端覆盖匿名点击星标打开认证弹窗、登录后加入/移除、会话过期清理和自选筛选结果。
- 手工验收：两个账号登录后自选列表彼此隔离；同一账号在另一浏览器登录后看到同一份自选。
- 部署验收：迁移不影响现有行情与 AI 路由；数据库位于稳定持久路径；Cookie 在 `/stock-macro/` 反向代理路径下可用。
